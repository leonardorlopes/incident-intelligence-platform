# ADR-001: Use LangGraph for Agent Orchestration

## Status
Accepted

## Context
The platform requires a robust way to manage complex, stateful workflows between multiple specialized agents. These workflows often involve cycles (e.g., if a retrieval result is insufficient, retry with different parameters) and conditional logic that standard linear chains cannot easily handle.

## Decision
We will use **LangGraph** (by LangChain) as the core orchestration framework for our Multi-Agent system.

## Rationale
- **State Management:** LangGraph provides built-in persistence and state management, allowing agents to share a common "memory" of the incident.
- **Support for Cycles:** Unlike standard DAG-based orchestrators, LangGraph allows for cyclic graphs, which are essential for iterative agent reasoning.
- **Granular Control:** It offers lower-level control over the flow compared to fully autonomous "Auto-GPT" style agents, which is necessary for predictable enterprise operations.
- **Ecosystem Integration:** Seamlessly integrates with the existing LangChain ecosystem for LLM wrappers and tool integrations.

## Consequences
- **Learning Curve:** Team members will need to understand the concept of state graphs and nodes.
- **Development Overhead:** Defining the graph structure explicitly adds some initial development time but provides long-term maintainability.
- **Vendor Lock-in:** We become tied to the LangChain/LangGraph ecosystem, though this is mitigated by the modular nature of the nodes.
