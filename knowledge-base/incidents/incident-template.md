# Incident Analysis Template

## Metadata
- **ID:** INC-YYYYMMDD-XXX
- **Severity:** [P1/P2/P3/P4]
- **Status:** [Active/Mitigated/Resolved]
- **Service:** [Service Name]
- **Reporter:** [Agent/User Name]

## Timeline
- **T0 (Detection):** When was the issue first noticed?
- **T1 (Acknowledge):** When did the team start investigating?
- **T2 (Mitigation):** When was the impact reduced?
- **T3 (Resolution):** When was the root cause fixed?

## Impact
- **Description:** What is the user-facing impact?
- **Scope:** How many users/regions are affected?
- **Data Loss:** Is there any potential for data loss?

## Root Cause Analysis (RCA)
- **Technical Root Cause:** Why did this happen technically?
- **Five Whys:**
  1. Why did X happen?
  2. Why did Y happen? (linked to 1)
  3. ...
- **Contributing Factors:** External events, config changes, etc.

## Resolution & Prevention
- **Fix Applied:** How was it solved?
- **Action Items:** List of Jira tasks to prevent recurrence.

- **Suggested Runbooks:** [Link to Runbook XYZ]
